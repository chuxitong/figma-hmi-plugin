# 必须同步到本机的文件清单

这份说明只讲一件事：

**云端改了哪些文件，你必须把这些文件同步到 Windows 本机，Figma 才会真的看到新版本。**

不同步 = Figma 永远报旧错，和云端改没改完全无关。

---

## 一、最少同步集（让 `catch {` 报错消失）

下面两个文件必须**一字不差**用云端版本覆盖本机：

1. **`figma-plugin/dist/code.js`** — 内含沙盒逻辑；若需 **Libraries 变量写入 `variables`（build ≥ `2026-05-07-r5-vars-bound-tree`）**，请确保该 `dist/code.js` 与仓库一致或本地执行 **`cd figma-plugin && npx tsc`** 由最新 **`src/code.ts`** 编译得到。  
2. **`figma-plugin/manifest.json`**

只有这两个到位之后：

## 二、推荐同步集（修复主插件全部已知问题）

如果你也想要拿到我之前做的“fetch 错误处理”、“预览安全注入”等修复：

3. **`figma-plugin/src/ui.html`**

这个文件是 UI 面板。  
不更新它，Figma 显示的还是旧 UI、旧错误处理（含 **Copy last API request** 的剪贴板兼容：`navigator.clipboard` 在插件 iframe 内常不可用，需同步新版本才会走 **execCommand 兜底 + 快照文本框**。Build 提示里 **`2026-05-07` 及以上**表示已包含该逻辑）。

---

## 三、可选同步集（只在你也想用 bootstrap 工具时才需要）

4. **`_localtools/figma-bootstrap/manifest.json`**  
5. **`_localtools/figma-bootstrap/ui.html`**

这两份是工具插件 `HMI Auto Bootstrap` 的最新版。  
新版主要差别：

- UI 里多了一个**SVG server URL 输入框**
- manifest 已与 Figma 当前规则兼容

如果你**不打算用** `figma-bootstrap`，这两份可以**完全不同步**，对主插件没有任何影响。

---

## 四、只在云端的服务端文件（**不**需要拉到本机）

如果你的服务端确实跑在云端，**下面这些不要拉到本机**，留在云端就好：

- `local-service/app.py`
- `local-service/renderer.py`
- `local-service/model_wrapper.py`
- `local-service/start_cloud_service.sh`（云端一键起服务；本机不设这个）
- `local-service/.venv/` 等

这些只要云端正确就可以。

---

## 五、怎么验证“本机这份文件”是不是真的新版本

### 验证 1：`figma-plugin/dist/code.js`

在 Windows 资源管理器里打开：

- `figma-plugin\dist\code.js`

用记事本搜：

```
catch
```

预期结果（新版）：

```
catch (_err) {
```

错误结果（旧版）：

```
catch {
```

只要你本机看到的是 `catch {`，说明这文件**没**同步过。

### 验证 2：`figma-plugin/manifest.json`

在 Windows 资源管理器里打开：

- `figma-plugin\manifest.json`

应该看到：

```json
"networkAccess": {
  "allowedDomains": ["*"],
  "reasoning": "...",
  "devAllowedDomains": [
    "http://localhost:8000",
    "http://localhost"
  ]
}
```

如果你本机和这一致，但 Figma 仍报旧错，请按下面流程让 Figma 真的重新读：

## 六、让 Figma 真的重新读 manifest 与 dist

1. **Plugins → Development**
2. 找到你导入的 `Figma HMI Plugin`
3. **Remove plugin**
4. 退出 Figma 桌面版，再开
5. **Plugins → Development → Import plugin from manifest…**
6. 一定要指向**步骤五里你刚刚用记事本核对过**的那份 `manifest.json`

如果你磁盘上有多份仓库副本（例如旧 zip 解压、`figma-plugin` 和 `figma-plugin_1` 两份），请确认你**Import 的就是新那份**。

---

## 七、关于 `localhost:8765`

- 主插件 `figma-plugin` **不会**访问 8765
- 只有工具插件 `_localtools/figma-bootstrap` 会
- 看到 `localhost:8765 ... ERR_CONNECTION_REFUSED`，多半意味着：
  - 你导入了 `figma-bootstrap`
  - 但本机或可访问范围内没有它要的素材服务

如果你**只想跑主插件 `figma-plugin`**，最简单办法是：

- 在 **Plugins → Development** 把 `HMI Auto Bootstrap` 也 **Remove**

只留主插件。

---

## 八、关于截图里的 `Quirks Mode`、`Ignored @property rule`、`Deprecated feature used`

这些来自 Figma 自身页面资源（`figma_app-...`、`vendor-core-...`）。

它们**不会**通过修改本仓库的 `manifest.json` / `dist/code.js` / `ui.html` 来消除。  
也不会影响主插件 `figma-plugin` 的功能。

---

## 九、一句话总结

- 云端这份代码：已修
- 你 Figma 看到的代码：除非同步，否则永远旧
- 同步是**你本机操作**，不是云端操作
