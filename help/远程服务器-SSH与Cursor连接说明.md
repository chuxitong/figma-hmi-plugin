# 用 Cursor 连上你的云服务器（写给第一次用的人）

---

## 这份文档是干什么的？

你在网上**租了一台 Linux 服务器**（跑模型、跑项目用的那台）。  
你想在 **Cursor 编辑器**里，像打开本机文件夹一样，**直接编辑那台服务器上的代码**。

要做成这件事，本质上就是：  
**让你的电脑，用「SSH」这条安全通道，连上服务器。**  
下面用大白话告诉你：**该点哪里、文件在电脑哪个角落、出错了大概是什么问题。**

---

## 先弄懂三个词（不用背，扫一眼就行）

| 词 | 人话 |
|----|------|
| **SSH** | 一种「远程登录」方式。你在自己电脑上敲命令，其实是那台服务器在执行。 |
| **密码登录** | 每次连都要输平台给你的 root 密码。能连，但麻烦，而且密码容易泄露。 |
| **密钥登录（推荐）** | 想像成一把**锁两瓣钥匙**：**私钥**永远只放在你自家电脑上；**公钥**复制到服务器上挂个号。以后电脑用私钥「对一下暗号」，就不用老输密码。 |

下面说的 **config、私钥、公钥**，都是帮你**少输密码、安全一点**用的。

---

## 你的电脑里，这些文件分别在哪儿？（Windows）

先找到你的用户目录：一般是 **`C:\Users\你登录 Windows 的名字`**。  
下面说的 **「用户目录下的 `.ssh` 文件夹」**，完整路径就是：

**`C:\Users\你的用户名\.ssh`**

打开方法（任选一种）：

1. 按 `Win + R`，输入 **`%USERPROFILE%\.ssh`**，回车；  
2. 或者打开「此电脑」→ 进 `C:\Users\你的用户名` → 找到 **`.ssh`**（若看不到，在「查看」里打开「显示隐藏的项目」）。

在这个 `.ssh` 文件夹里，你会常见到这些文件（名字可能因配置略有不同）：

| 文件（大致名字） | 是干啥的？（人话） |
|------------------|-------------------|
| **`config`** | 一个**小通讯录**：给服务器起个**简称**（比如 `seetacloud-westc`），里面写好真实地址、端口、用哪个用户、用哪把私钥。以后你打字就用简称，不用记一长串。 |
| **`id_ed25519_seetacloud`（没有后缀的）** | **私钥 = 你家的门钥匙。千万不能发给别人、不要传到 Git 网盘、不要发聊天里。** |
| **`id_ed25519_seetacloud.pub`（有 .pub 的）** | **公钥 = 可以贴到服务器上的那一半。** 给服务器看这张「纸条」，它就知道允许哪台电脑免密连上来。 |
| **`known_hosts`** | 第一次连某台机器时，电脑记一下「这家的长相」，防被骗连到假服务器。**一般不用你动。** |

**一句话记：**  
- **没 `.pub` 的** = 私钥，**保密**；  
- **有 `.pub` 的** = 公钥，**可以放到服务器**。

## 在 Cursor 里要装什么？

1. 打开 **Cursor**；  
2. 按 **`Ctrl + Shift + X`** 打开「扩展」；  
3. 搜 **Remote SSH** 或 **Remote - SSH**；  
4. 点**安装**。

装好后，按 **`F1`**（或 `Ctrl + Shift + P`），在输入框里打：**`Connect to Host`**，选 **`Remote-SSH: Connect to Host...`**。  
在列表里选你在 **`config` 里起好的那个简称**（例如文档里曾用过的 `seetacloud-westc`）。

若已经配好**公钥登录**，多半**不会再要密码**；若第一次还弹密码，可能是公钥没配对成功，见文末「常见现象」。

---

## 连上之后，项目文件夹放哪？（以 AutoDL 类机器为例）

云厂商给你的 Linux 里，常有一个专门放**数据、大文件**的盘，路径类似：

**`/root/autodl-tmp/`**

**建议**把整个 **`figma-hmi-plugin` 项目**（你本机 `D:\cursorwork\figma-hmi-plugin` 那一套）**复制/上传到**：

**`/root/autodl-tmp/figma-hmi-plugin`**

然后在 Cursor 里：**「文件」→「打开文件夹」** → 选这个路径。  
这样代码和以后下载的**大模型**都容易跟数据盘放一起，不挤系统盘。  
项目里更细的**装环境、开服务**在仓库的 **`docs/DEPLOYMENT_AUTODL.zh.md`**，那是「下一步」的事。

---

## 服务器上跟密钥有关的，就一个文件要心里有数

在**服务器**上（不是你家 Windows），root 用户通常在这里登记「哪几把公钥能免密登录」：

**`/root/.ssh/authorized_keys`**

里面**一行 = 一把公钥**（从你家 `.pub` 文件里复制来的那一长串）。  
**你一般不用手改**；知道「我把我电脑的公钥加进这里 = 我电脑能免密连」就够。

## 想自己试一下「到底连上没」？（可选）

在 **PowerShell** 里输入（`seetacloud-westc` 要换成你 `config` 里写的那个简称，若你还没配置简称，这行可能不适用，先做上面的 Cursor 连接即可）：

```text
ssh seetacloud-westc
```

能进去、出现 Linux 的提示符，就说明**通道是通的**。

如果已经配好公钥、不想输密码，也可以让系统帮你测**「是不是免密成功」**（有基础再试，看不懂可跳过）：

```text
ssh -o BatchMode=yes -o ConnectTimeout=20 seetacloud-westc "echo ok"
```

能打出 `ok` 且**没问你要密码**，说明**密钥这一步是对齐的**。

## 安全提醒（一定要看，很短）

- **私钥**（没有 `.pub` 的那个）= 家门钥匙，**别截图、别发群、别上传 Git**。  
- **根密码**若曾在不安全的地方说过，**去云平台里改掉**，以后尽量只用密钥。  
- 换电脑时：在新电脑重新生成或拷贝密钥、再把**新公钥**加到服务器 `authorized_keys`；旧电脑不再用就当钥匙作废处理。

---

## 常见现象（人话版）

| 你看到的 | 可能是什么意思 |
|----------|------------------|
| 一直要输密码 | 公钥没配对成功，还在走密码登录。检查「本机 `config` 里私钥路径对不对」「服务器 `authorized_keys` 里有没有你这台电脑的 `.pub` 整行」。 |
| 连不上、超时 | 云服务器没开机、实例关掉了、或者平台给你的**地址/端口**变了。回控制台看一眼再改 `config`。 |
| 提示 `Permission denied (publickey)` | 基本就是「私钥/公钥对不上」。对照上面两行检查。 |
| Cursor 里连上了，但打开文件夹是空的 | 你还没把项目上传到服务器，或者路径打错了。确认 **`/root/autodl-tmp/figma-hmi-plugin` 里真有文件**再打开。 |
