# 断联前准备与换机后展开项目

适用于：**即将关闭/释放 AutoDL 等远程实例**、或 **下一台新服务器上从零拉起** 同一仓库。你换机器后，**本机 Figma 与 Windows 工作流不需要改**；要重建的是**云端侧** Python 服务与（可选）模型权重。

---

## 一、断联前建议做完的事（10 分钟）

1. **代码与文档进 Git（强烈推荐）**  
   在服务器仓库根目录：

   ```bash
   cd /root/autodl-tmp/figma-hmi-plugin
   git status
   # 确认需要保留的修改都已 add/commit；未纳入 Git 的本地改动请另存
   git push
   ```

   若没有远程仓库，至少：

   ```bash
   tar czvf ~/figma-hmi-plugin-snapshot-$(date +%F).tgz --exclude=node_modules --exclude=local-service/.venv --exclude=.git 'figma-hmi-plugin' 2>/dev/null
   # 路径请按你实际仓库父目录调整；大文件/模型见下文
   ```

2. **记下环境变量**（下一块机器上还要用）  
   在常用 shell 里曾 export 的项，存成**纯文本**（可放在个人笔记或 `~/.hmi_env.local`，**勿**提交含密钥的文件）：

   - `USE_REAL_MODEL=1`
   - `UI2CODEN_QUANT=none`（全精度，与论文/指标一致）
   - `UI2CODEN_DEVICE_MAP=single` 或 `auto`
   - `UI2CODEN_MODEL_ID=/你的路径/UI2Code_N`（**新机器上路径会变的，只记变量名和含义**）
   - 若用过：`HF_ENDPOINT`、`PLAYWRIGHT_GOTO_TIMEOUT_MS` 等

3. **记下「对外怎么连 8000」**  
   - 用 **SSH 本地转发**时，你在本机执行的完整命令是：  
     `ssh -p 10029 -L 8000:127.0.0.1:8000 root@connect.westd.seetacloud.com -N`  
     在新开的终端里保持不断开；Figma 插件里 **Service URL 仍填** `http://127.0.0.1:8000` 或 `http://localhost:8000`。换到另一台云实例后，用新控制台上「整条 ssh 登录命令」里的端口、用户、主机替换本行中的对应段，并同步改 `help/本地浏览器访问云端8000-SSH隧道.zh.md` 与 `local-service/start_cloud_service.sh` 里相同内容。  
   - 用平台 **公网/HTTPS 反代** 时：记下**完整 base URL**（与 `manifest` 的 `networkAccess` 一致即可），插件里改 **Service URL** 为该地址。

4. **不必强行拷贝的内容（体积大、可再生成）**  
   - `local-service/.venv`：新机器上 `python -m venv` 重装即可。  
   - `**/.venv`、`node_modules`：可重装。  
   - **UI2Code^N 权重**（常十几 GB）：新机器上按 `help/docs/DEPLOYMENT_AUTODL.zh.md` 再 `snapshot_download` 一次，或从对象存储/你方 NAS 拉取同一目录。

5. **本机 Figma 侧**  
   无需在服务器上操作；**下次开发**时仍用 `help/必须同步到本机的文件清单.md` 从**新拉下来的仓库**里同步 `figma-plugin` 到 Windows。

## 二、新服务器上「直接展开项目」的最短路径

假设已通过 **git clone** 或解压拿到了 **与当前结构一致** 的 `figma-hmi-plugin` 目录（路径可自定，下文用 `REPO` 表示）。

1. **Python 与依赖（渲染 + API）**  

   ```bash
   cd REPO/local-service
   python3 -m venv .venv
   source .venv/bin/activate
   pip install -U pip
   # PyTorch 按机器 CUDA 选轮子，见 help/远程服务器-上传后补装依赖.md
   pip install -r requirements.txt
   playwright install chromium
   # 若缺系统库再执行: playwright install-deps
   ```

2. **大模型目录**（与 `UI2CODEN_MODEL_ID` 一致）  
   按 **`help/docs/DEPLOYMENT_AUTODL.zh.md` §四** 下载或解压到**本机可写路径**，并设置：

   ```bash
   export UI2CODEN_MODEL_ID=/新路径/UI2Code_N
   export USE_REAL_MODEL=1
   export UI2CODEN_QUANT=none
   export UI2CODEN_DEVICE_MAP=single
   python verify_model_install.py
   ```

3. **启动服务（监听 0.0.0.0 便于转发）**  

   ```bash
   cd REPO/local-service
   source .venv/bin/activate
   python -m uvicorn app:app --host 0.0.0.0 --port 8000
   ```

4. **健康检查**  

   ```bash
   curl -s http://127.0.0.1:8000/health | python -m json.tool
   ```

   期望见 `ui2code_n_active: true`（模型就绪后）。

5. **从笔记本连过来**  
   在本机开 **SSH 转发**并保持该窗口不关：

   ```bash
   ssh -p 10029 -L 8000:127.0.0.1:8000 root@connect.westd.seetacloud.com -N
   ```

   换实例后整条命令要与新控制台 SSH 口令一致（见 **`help/本地浏览器访问云端8000-SSH隧道.zh.md`**）。  

   浏览器与 Figma 仍访问 `http://127.0.0.1:8000`。

## 三、和现有文档的对应关系（不必重复记）

| 需求 | 文档 |
|------|------|
| AutoDL 全量部署、显存、HF 镜像、双卡、Blackwell 等 | `help/docs/DEPLOYMENT_AUTODL.zh.md` |
| SSH / Cursor 连服务器 | `help/远程服务器-SSH与Cursor连接说明.md` |
| 上传后补装 PyTorch/依赖 | `help/远程服务器-上传后补装依赖.md` |
| 插件、本机要同步哪些文件 | `help/必须同步到本机的文件清单.md` |
| 插件不响应、自测、URL 等 | `help/插件没反应-本机同步与自测.md` |

换服务器后，**仅**「服务器 IP/SSH/权重路径/环境变量」会变；**仓库内路径**、**Figma 插件**、**`baseline-tests` 跑批** 的用法**不变**。

---

## 四、你换实例时容易忘的两点

1. **新机器上** `UI2CODEN_MODEL_ID` **必须**指向**实际存在的**模型目录。  
2. **uvicorn 必须** `--host 0.0.0.0`，否则从隧道/公网**打不到** 8000。
