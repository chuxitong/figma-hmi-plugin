# 下次重连 · 快速接续

写给从本机再次用 SSH / Cursor 打开**同一台远程机器**时的自己。

## 1. 怎么连回来

连接步骤、密钥与 Cursor 远程文件夹：见 **`help/远程服务器-SSH与Cursor连接说明.md`**。  
AutoDL / 依赖与 GPU：见 **`help/docs/DEPLOYMENT_AUTODL.zh.md`**、**`help/远程服务器-上传后补装依赖.md`**。

## 2. 项目在服务器上的路径

```text
/root/autodl-tmp/figma-hmi-plugin
```

用 Cursor **Open Folder** 打开上述目录即可继续。

## 3. 常用启动命令（按需）

**本地 HTTP 服务（云端跑 uvicorn）：**

```bash
cd /root/autodl-tmp/figma-hmi-plugin/local-service
# 若已有 venv：source .venv/bin/activate
pip install -r requirements.txt   # 首次或依赖变更
bash ./start_cloud_service.sh
```

或手动：`uvicorn app:app --host 0.0.0.0 --port 8000`。

**在本机新开一个终端**，保持不关，让整个电脑上的 **`127.0.0.1:8000` 通向云上服务**：

```powershell
ssh -p 10029 -L 8000:127.0.0.1:8000 root@connect.westd.seetacloud.com -N
```

然后本机浏览器打开 `http://127.0.0.1:8000/docs`。仅登录云上（不做端口映射）时用：`ssh -p 10029 root@connect.westd.seetacloud.com`。更细的说明见 **`help/本地浏览器访问云端8000-SSH隧道.zh.md`**。

**Figma 插件（改完 `src/code.ts` 后）：**

```bash
cd /root/autodl-tmp/figma-hmi-plugin/figma-plugin
npm install   # 首次
npm run build
```

在 Figma → Plugins → Development 里重新加载插件；**Service URL** 远程时记得端口映射或 `allowedDomains`。

**本实例 Node（接续已处理）：** 若 `node` / `npm` 报 `sqlite3session_attach`，说明 conda 的 `libnode` 与系统 sqlite 冲突。已在 **`/opt/node-v20.18.2`** 安装官方 Node 20；新终端请先执行 `export PATH="/opt/node-v20.18.2/bin:$PATH"`，或依赖已在 `~/.bashrc` 末尾加入的同名 `PATH` 前缀（交互式 shell 会自动生效）。

## 5. 环境说明

当前工作区**可能未初始化 git**（仅拷贝/同步目录时）。若你需要版本管理，可在项目根目录自行 `git init` 或从远端克隆同名仓库后再对齐文件。

## 6. 断开前你可以自己做的一步（可选）

- 若在跑 **uvicorn**，终端里 `Ctrl+C` 停掉，避免占端口。  
- 若有未保存的编辑，在 Cursor 里保存全部文件（Ctrl+K S 或菜单 Save All）。
