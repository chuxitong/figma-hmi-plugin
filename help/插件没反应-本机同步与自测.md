# Figma 插件「点 Generate 没反应」— 本机同步与自测（最权威的一份）

> 本文不重复猜测，目标只有两件事：  
> 1. **让本机 Figma 实际加载到的就是仓库里最新的 `figma-plugin`**；  
> 2. **不依赖任何网络请求**，先确认「点击 → 沙箱」消息通道本身是通的。

如果这两件事都验证通过，再回到 Generate 上看是「选区」「Service URL / 隧道」「服务侧」哪一段。

---

## 一、最近一版插件的「指纹」（你必须看到下面这些）

最新一版 `figma-plugin` 编进去了一个 **构建标记** 与一个 **不联网的自测按钮**。

把仓库里 **这两个文件** 拷到本机的 `figma-plugin` 目录后再重新导入插件：

| 仓库路径（云端 / 服务器） | 本机路径（Windows） |
|---|---|
| `figma-plugin/manifest.json` | `<本机克隆路径>\figma-plugin\manifest.json` |
| `figma-plugin/src/ui.html` | `<本机克隆路径>\figma-plugin\src\ui.html` |
| `figma-plugin/dist/code.js` | `<本机克隆路径>\figma-plugin\dist\code.js` |

打开本机的 **`figma-plugin/dist/code.js`**，确认前 5 行包含：

```text
const BUILD_STAMP = "2026-04-28-r3-sandbox-js";
figma.notify(`HMI plugin build ${BUILD_STAMP} loaded`, { timeout: 3 });
```

打开本机的 **`figma-plugin/src/ui.html`**，搜索一次以下三处（任一缺失就说明你拿到的不是最新）：

- `Build: 2026-04-28-r3b-service-url-rewrite`（`getServiceUrl` 会**规范化并回写**输入框与存储，避免 `8000。` 进入 `fetch`）
- `Run Self-Test (no network)`
- `function storageGet`（`localStorage` 安全包装，避免 data: 环境下整页脚本挂掉）

若没看到，**说明你本机磁盘上的文件不是最新仓库版**——先把上面三个文件覆盖过来再继续。

## 二、在 Figma 里强制重载插件

> Figma（无论桌面版还是网页版）会**缓存**已 import 的开发插件。**仅修改文件不会生效**，必须按下面顺序：

1. **关闭** 当前文件里打开的本插件面板。  
2. 顶部菜单 **Plugins → Development → Manage plugins in development**。  
3. 找到「Figma HMI Plugin」一行，点 **Remove**（彻底移除，不只是关闭）。  
4. 再 **Plugins → Development → Import plugin from manifest…**，选刚才本机已同步的 **`figma-plugin/manifest.json`**。  
5. 重新通过 **Plugins → Development → Figma HMI Plugin** 打开面板。

打开瞬间应该能在 **画布底部** 看到一条 Figma 系统 Toast：

> `HMI plugin build 2026-04-28-r3-sandbox-js loaded`

**Toast 是从主线程里 `figma.notify` 发的**，与 `ui.html` 是否新版无关——Toast 出现，说明你的 `dist/code.js` 是新版；不出现，则 `dist/code.js` 还是旧的。

面板最底部 **Log** 下方一行也会出现蓝字：

> `Build: 2026-04-28-r3b-service-url-rewrite ...`

这一行是从 `ui.html` 渲染的——这条出现，则 `ui.html` 是新版。

> **两条都出现，再继续往下；任意一条缺失，都不要继续 —— 先把对应文件再同步一次重新 Remove + Import。**

## 三、第 1 步：完全不依赖网络的「点击通道」自测

新版面板里加了一个 **紫色按钮 `Run Self-Test (no network)`**，就在 Generate Code 下面。

点它一次。**预期同时发生**：

1. 紫色按钮的文字立刻变为 `CLICKED HH:MM:SS — sending…`，2 秒后还原。  
2. 画布底部出现 Figma 系统 Toast：`HMI: self-test received from UI (build 2026-04-28-r3-sandbox-js).`  
3. 面板底部 **Log** 至少新增两行：
   - `Self-Test: UI click ok → postMessage 'self-test' (no network).`
   - `Self-test OK: sandbox responded (build 2026-04-28-r3-sandbox-js).`  

### 现象 A：三件都出现 ✅

「UI ↔ 沙箱」消息通道**完全没问题**，本机插件加载也是新版。  
此时 Generate 仍不反应只可能是后续两类问题：**选区** 或 **Service URL / 隧道 / 服务**。直接跳到 §五。

### 现象 B：按钮文字会变，但 Toast 与 Log 都没新增 ⚠️

`ui.html` 已被 Figma 加载，但 `dist/code.js`（沙箱主线程）**没收到消息**。多见于：

- `dist/code.js` 不是最新（沙箱里没有 `self-test` 处理分支）；  
- 你 import 的是另一个目录里的 `manifest.json`（指向旧 `dist/code.js`）。

**对策**：把 `dist/code.js` 再次同步到 import 时指向的目录，重新 Remove + Import。

### 现象 C：连按钮文字都不变 ⛔

JS 没运行到 `onclick`，多半是：

- `ui.html` 没换成新版（旧版没有这个按钮和这段 `flashClickedLabel`）；  
- 你的 Figma 还是缓存的旧 UI（没 Remove 直接 Reimport 时偶发）。  

**对策**：完全 Remove 这个 dev plugin，关闭 Figma，重新打开 Figma，再 Import。

### 现象 D：紫色按钮在，也点了，但像「没反应」——先看成有没有 **CLICKED … — sending…**

- **有** 这行字闪一下（2 秒左右还原）：说明 `onclick` 和 `log` 都执行了。若 **Log 仍停在** `…postMessage 'self-test'…` 且 **没有** 后面的 *Self-test OK* 与 **没有** 画布 Toast，多半是 `parent.postMessage` 的 **第二参数** 在 **Figma 桌面版** 下与 `https://www.figma.com` 不匹配，**UI → 主线程** 消息没落下来。本仓库已把 `ui.html` 里 `postToSandbox` 改成 **与官方示例一致**（只发 `{ pluginMessage: … }`，并优先用 `'*'`，与旧版里额外带 `pluginId` 的写法不同）——**请把最新 `figma-plugin/src/ui.html` 再同步到本机后 Remove + Import 一次。**
- **连** `CLICKED` **都没有，Log 也完全不增加第 1 行 Self-Test 文案**：在 **Plugins → Development → Show/Use Console** 里看界面 iframe 的 **红色报错**（有错误时后续 `getElementById("btnSelfTest").onclick=…` 可能根本没执行）。

> 这三种情况和「Generate 不反应」的根因是一一对应的；先把 Self-Test 的现象稳定下来，再看 Generate。

## 三点五、主线程里若出现 `Syntax error … Unexpected token`（与插件无关的 Figma 日志可忽略）

在 **Show/Use Console** 里，若**只有** Figma 自己的 `Statsig` / `vendor-*.min.js` 之类报错，**可以忽略**。

若**同时**出现**指向你插件 `code.js`** 的句法错误，以前常见原因有两类，**r3 已修**：

1. **可选链** `msg.payload?.foo`：Figma 插件**主线程**里跑的 JS 引擎**不接受**经 TS 转译后仍带 `?.` 或等价的 `void 0` 链的写法，会导致整份 `figma.ui.onmessage` **根本没注册**，点按钮**全无反应**。（仓库已改为 `const pay = msg.payload; pay && pay.xxx` 等显式判断，并把 `tsc` 的 `lib` 收紧为 `ES2017` 再产 `dist/code.js`。）  
2. **面板 iframe** 在部分环境下**禁用 `localStorage`**（Console 里 `Storage is disabled inside 'data:' URLs`）：一加载就会抛错，后面 `onclick` 都绑不上。`ui.html` 已用 `try/catch` 包一层 `storageGet` / `storageSet`。**务必将 `src/ui.html` 与 `dist/code.js` 一并同步。**

修完后务必 **Remove + Import**，并确认 `BUILD_STAMP` 为 **`2026-04-28-r3-sandbox-js`**。

## 四、第 2 步：用 DevTools 看真正的 console（极快定位 JS 错误）

Figma 桌面版/网页版打开开发插件后，菜单 **Plugins → Development → Show/Use Console**（或 macOS 的 `Cmd+Option+I`，Windows `Ctrl+Shift+I`）打开 DevTools。

切到 **Console** 选项卡再点 **Self-Test**：

- 应能看到沙箱里的 `INFO` 级日志（比如「postMessage to sandbox」）。  
- 若 Console 里有 **红色 `Uncaught` / `SyntaxError` / `ReferenceError`**，把那一行截给我；那是脚本顶部就抛错，所有 onclick 都不会绑定。

## 五、第 3 步：再点 Generate（仅当 Self-Test 已通过）

如果 §三现象 A 已稳定通过，再来 Generate。完成下面三件事再点：

1. **左侧 Layers** 里点中**最外层**那一行 Frame（不是某个内部 group / 文本），例如 `04-operator-panel` 或 `05-production-overview`。  
2. **Service URL** 已 **Save URL**：服务在远端 + SSH 转发时，仍然填 `http://127.0.0.1:8000`；Save 完看 Log 里有 *Service URL saved: http://127.0.0.1:8000*。  
3. **隧道** 这一头活着：本机 PowerShell 里的 `ssh -L 8000:127.0.0.1:8000 ...` 会话**没断**（窗口别关），并且 Edge / Chrome 打开 `http://127.0.0.1:8000/health` 能拿到 `ui2code_n_active: true`。

满足这三件事再点 Generate。预期：

- 蓝色按钮文字一闪：`CLICKED HH:MM:SS — sending…`。  
- 画布底部 Toast：`HMI: exporting "..."… (may take 10–40s for large frames)`。  
- Log 依次出现：  
  `Generate: UI click ok → postMessage to sandbox…`  
  `Sandbox: exporting "..." as PNG @2x …`  
  `Frame "..." exported (W×H).`  
  `Sending to local service for code generation...`  
- 真模型大约 **90–150 秒** 才会回写 `Code generation complete.`，请耐心等。

如果只看到前两行就停住，是**导出环节卡住**（一般大画框首次有点慢，或 `getCSSAsync` 在某些 Figma 版本里偶尔卡）；30 秒以上还没第三行，说明导出真的失败，看下 Toast 里是不是 *export failed*。

如果出现 `Service error: ...`，把后面那段错误原文贴出来，那才是网络/服务/CORS 这一线的问题。

### 若错误为 `Failed to parse URL from http://127.0.0.1:8000。/generate`

Service URL 输入框里**末尾多了一个中文句号 `。`**（或全角点），拼出来变成 `...8000。/generate`，`fetch` 会报「无法解析 URL」。**删掉句末 `。` 后点 Save URL**，或同步最新 `ui.html`：我们会自动剥掉 `。` / 零宽字符并写回输入框。

## 六、关于「Windows 是不是问题」

不是。`figma-plugin` 全部代码、`ssh -L` 转发、Figma 桌面版 / 网页版，都在 **你 Windows 本机**就能跑。后端 `uvicorn` 在 **AutoDL 服务器**上，没有把任何 Linux 命令塞进 Windows，**架构没变**。详见 `help/docs/DEPLOYMENT_AUTODL.zh.md` 第六、八节。
