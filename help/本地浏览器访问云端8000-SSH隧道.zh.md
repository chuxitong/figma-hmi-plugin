# 本机浏览器打开 `http://127.0.0.1:8000/docs` 访问**云端**服务

## 结论先说清

- **uvicorn 跑在云端 Linux 上**，监听 **`0.0.0.0:8000`**（见 [`help/docs/DEPLOYMENT_AUTODL.zh.md`](docs/DEPLOYMENT_AUTODL.zh.md) §五）。
- **你 Windows 上的 `127.0.0.1:8000` 默认就是你自己电脑**，不会自动连到云端。
- 要让本机地址连到云上 8000，必须在 **你自己电脑** 上执行下面的 **SSH 本地端口转发**（另开一类窗口，`Ctrl+C` 或关窗即断开）。
- 这条命令**只能在你自己的终端执行**；云端无法替你在本机点开隧道。

---

## 在你自己的 PowerShell / 终端里（完整可运行）

登录（不设转发，正常进 shell）：

```powershell
ssh -p 10029 root@connect.westd.seetacloud.com
```

本机 **`127.0.0.1:8000` → 云上 `127.0.0.1:8000`**（只转发、不占用一个交互 shell，`窗口要一直开着`）：

```powershell
ssh -p 10029 -L 8000:127.0.0.1:8000 root@connect.westd.seetacloud.com -N
```

- **`-L 8000:127.0.0.1:8000`**：本机端口 8000 经 SSH 指到远端回环上的 uvicorn。
- **`-N`**：不执行远程命令，只做端口转发。

然后在本机 Chrome / Edge 打开：

```text
http://127.0.0.1:8000/docs
http://127.0.0.1:8000/health
```

若平台将来给你的 SSH **端口或主机名**换了，只要把上面两条 `ssh` 里 **整条** `10029`、`connect.westd.seetacloud.com`、`root` 换成新控制台上的值。

---

## 与 Figma 插件一致

插件 **Service URL** 填 **`http://127.0.0.1:8000`** 或 **`http://localhost:8000`**（**不要**中文句号 `。`）。前提：**本机已执行带 `-L` 的那条 SSH 并保持连接**。

---

## 云端侧你必须已经有的

1. 服务器上已启动：`uvicorn app:app --host 0.0.0.0 --port 8000`
2. 在**服务器本机**上能通：`curl -s http://127.0.0.1:8000/health`

在**服务器的 SSH 会话**里可一键拉起（预热真模型、日志默认 `/tmp/uvicorn-figma-hmi.log`）：

```bash
bash /root/autodl-tmp/figma-hmi-plugin/local-service/start_cloud_service.sh
```

---

## 常见故障

| 现象 | 处理 |
|------|------|
| `ssh: connect ... Connection refused` | 控制台是否换了端口/域名；是否与当前实例仍未释放有关。 |
| 本机能开 `/docs`，Figma 里 fetch 失败 | 看 [`help/docs/DEPLOYMENT_AUTODL.zh.md`](docs/DEPLOYMENT_AUTODL.zh.md) §八（CORS / Private Network）；`local-service/app.py` 已处理 PNA。 |
| 隧道建了仍打不开页面 | 云上 uvicorn 未起或未监听 8000；查 `tail -f /tmp/uvicorn-figma-hmi.log`。 |
