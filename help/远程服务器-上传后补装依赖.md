> 你本次打包在 **`D:\cursorwork\figma-hmi-plugin-for-upload.zip`**，已通过 `scp` 传到  
> **`/root/autodl-tmp/figma-hmi-plugin-for-upload.zip`**，并解压到  
> **`/root/autodl-tmp/figma-hmi-plugin`**。  
> 打包时**故意未包含**（需在服务器上重装）：`figma-plugin/node_modules`、`local-service/.venv`、各处 `__pycache__`、**`.git`**（减少体积）。

## 2. 在远程**终端**里要执行的命令（按顺序）

以下假设你是 **root**，系统带 **miniconda**（路径常见为 `/root/miniconda3`）。若你机器上命令略有不同，以 `which python` 为准。

### （1）装 Node（打 Figma 插件用）

本镜像若**没有** `node` / `npm`，先装一种即可（**任一方式成功即可**）：

**方式 A：用 apt（简单）**

```bash
apt-get update && apt-get install -y nodejs npm
node -v
npm -v
```

若 `npm -v` 报 **Cannot find module 'semver'** 等错误，说明当前镜像里 apt 的 `npm` 损坏，请改用 **方式 C**。

**方式 B：用 conda（若你更习惯 conda）**

```bash
source /root/miniconda3/etc/profile.d/conda.sh
conda install -c conda-forge -y nodejs
node -v
npm -v
```

若 `node` 报 **`sqlite3session_attach` 符号未找到**，说明 conda 的 `libnode` 与系统 sqlite 冲突，请改用 **方式 C**。

**方式 C：官方 Node 二进制（推荐在 A/B 失败时使用）**

```bash
cd /tmp && curl -fsSL https://nodejs.org/dist/v20.18.2/node-v20.18.2-linux-x64.tar.xz -o node.tar.xz \
  && tar -xf node.tar.xz
export PATH="/tmp/node-v20.18.2-linux-x64/bin:$PATH"
node -v && npm -v
```

之后执行 `npm install` / `npm run build` 前请保持该 `PATH`（或把该目录加入 `~/.bashrc`）。

### （2）安装 figma 插件依赖并编译

```bash
cd /root/autodl-tmp/figma-hmi-plugin/figma-plugin
npm install
npm run build
```

### （3）为 local-service 建 Python 虚拟环境并装依赖

用 miniconda 自带的 Python 建 venv 即可：

```bash
cd /root/autodl-tmp/figma-hmi-plugin/local-service
/root/miniconda3/bin/python -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
```

**PyTorch / CUDA（务必按显卡选择）**

- **NVIDIA Blackwell（如 RTX 5090，驱动里 CUDA 13.x，计算能力 sm_120）**：**不要**装 `cu124` 的 PyTorch，否则推理会报 *no kernel image* 或架构警告。应安装 **cu128** 官方轮子，再装其余依赖，例如：

```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu128
pip install -r requirements.txt
pip install accelerate bitsandbytes huggingface_hub
```

- **Ada Lovelace / 24 GB 级云 GPU（如 RTX 4090、A10）**：可与仓库 `help/docs/DEPLOYMENT_AUTODL.zh.md` 一致，使用 `cu124`：

```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124
pip install -r requirements.txt
pip install accelerate bitsandbytes huggingface_hub
```

国内拉 Hugging Face 模型较慢时，下载权重前可设：`export HF_ENDPOINT=https://hf-mirror.com`。

### （4）Playwright（/render 与预览需要）

```bash
# 仍在 local-service 且已激活 .venv
playwright install chromium
playwright install-deps
```

若 `install-deps` 报错，按终端提示在 Ubuntu 上装缺的系统库后重试。

### （5）与 AutoDL 文档一致时：准备跑真实模型

权重需自行下载，例如到 `/root/autodl-tmp/UI2Code_N`，再设环境变量（**示例**）：

```bash
export USE_REAL_MODEL=1
export UI2CODEN_QUANT=none
export UI2CODEN_MODEL_ID=/root/autodl-tmp/UI2Code_N
export UI2CODEN_DEVICE_MAP=single
```

再启动服务（需 `--host 0.0.0.0` 才能公网/映射访问）：

```bash
cd /root/autodl-tmp/figma-hmi-plugin/local-service
source .venv/bin/activate
python -m uvicorn app:app --host 0.0.0.0 --port 8000
```

更细的说明见仓库内 **`help/docs/DEPLOYMENT_AUTODL.zh.md`**。

## 3. 打包时忽略了什么？要不要「补齐」`.git`？

| 已忽略 | 在远程要不要补 |
|--------|----------------|
| `node_modules` | **要**：见上 `npm install` |
| `local-service/.venv` | **要**：见上 `python -m venv` + `pip` |
| `__pycache__` | **不用**：跑 Python 会自动生成；可删可留 |
| `.git` | **一般不用**：在服务器上写代码/跑实验不必有 Git 历史。若你**非要用 Git**，可在**远程**里 `git clone` 你仓库，或本机多打一个**含 .git 的包**再传（会更大） |
