本文说明：**第四周任务书里的要求如何在当前仓库中落实**、**各交付物是什么、放在哪里**、以及**您如何一步步查看与验证**（尽量写得直白，适合第一次接触本仓库的读者）。

## 一、第四周任务书在说什么（对照中文翻译稿）

依据 **`help/张思成-任务计划（中文翻译）.md`** 中「第四周」一节，核心要求是：

1. **把基线模型做成一个本地 HTTP 服务**  
   Figma 插件不直接在插件里跑大模型，而是请求本机（或同一台机器上）的另一个进程。

2. **服务应接受**（与翻译稿原文对齐；「可选」= 协议须支持，单次请求可省略）  
   - 界面的 **PNG**（传输上一般为 Base64）  
   - **任务描述或提示词（prompt）**（字段 `task_description`；编辑流为 `instruction`）  
   - （可选）**当前代码** `current_code`（精修/编辑迭代用；由插件保存模型输出，非 Figma 导出）  
   - （可选）**来自 Figma 的变量** `variables`、**CSS 提示** `css_hints`（插件勾选是否附带）  

3. **服务应返回**（同上，「可选」= 字段可空，但服务与客户端须支持）  
   - **生成或更新的 HTML/CSS**（`code`）  
   - （可选）**简短说明**（`explanation`）  
   - （可选）**下一阶段的元数据**（`metadata`）  
   - 说明：可选返回是否在面板里显示，由 **`ui.html`** 中 **Show explanation / Show metadata / Show preview image** 勾选；服务端仍会返回对应 JSON 字段。可选请求字段（任务描述、变量、CSS、当前代码、`instruction`）均由 **Include …** 类勾选框决定是否写入当次 HTTP 请求，见 **`help/docs/figma-data-extraction.zh.md`**。  

4. **API 形态**  
   任务书允许「多个端点」或「统一 API」。**本仓库采用多个端点**：生成、精修、按指令编辑、纯渲染等分开，便于插件按场景调用。

5. **交付物**（任务书原文精神）  
   - 可运行的服务代码  
   - 简短的 API 说明（含请求/响应示例）  
   - 一份报告：**至少包含一次完整的演示请求与响应**（或等价说明）

> **关于 `help/Чжан Сычэн.docx`**：该文件是 Word 二进制格式，在多数开发环境里无法像 Markdown 一样直接「全文搜索」。第四周的具体条目请以 **`help/张思成-任务计划（中文翻译）.md`** 为准；若 `.docx` 与翻译稿不一致，以您导师下发的最新版任务书为准。

## 三、如何在本机「跑起来」服务（逐步）

### 3.1 环境前提

- 已安装 **Python 3.10+**（与仓库 `README` / `local-service/requirements.txt` 一致即可）。  
- 建议在项目根目录为 `local-service` 单独建虚拟环境并安装依赖（具体命令见仓库 **`README.md`** 或 **`help/docs/DEPLOYMENT_AUTODL.zh.md`** 若您在 AutoDL 上部署）。

### 3.2 安装依赖（典型命令）

在终端中进入 **`local-service`** 目录，创建并激活 venv，然后：

```bash
pip install -r requirements.txt
```

若您要跑 **真实 UI2Code^N 模型**，还需按仓库说明安装 **PyTorch（CUDA）**、**Playwright 浏览器**、以及下载权重等（见 **`README.md`** 与 `scripts/download_ui2code_n_weights.py` 等脚本说明）。  
若暂时不装 GPU 模型，服务仍可启动，但可能在首次请求时回退到规则/占位实现（行为见 `help/docs/api-reference.zh.md` 中「服务内部使用的模型」一节）。

### 3.3 启动服务

在 **`local-service`** 目录下：

```bash
uvicorn app:app --host 0.0.0.0 --port 8000
```

- **`localhost:8000`**：本机浏览器、本机 Figma 插件默认连这个地址即可。  
- **`0.0.0.0`**：若服务跑在远程服务器（如 AutoDL），需要对外网卡监听时用这个 host（安全组/防火墙还需放行端口）。

### 3.4 确认服务活着

浏览器打开：

```text
http://localhost:8000/health
```

或远程则把 `localhost` 换成服务器 IP。  
若返回 JSON 且含 `status` 等字段，说明进程正常。字段含义见 **`help/docs/api-reference.zh.md`**。

### 3.5 用浏览器查看「交互式 API 文档」（推荐）

FastAPI 自带 Swagger UI，在浏览器打开：

```text
http://localhost:8000/docs
```

您可以直接在页面上点开 **`POST /generate`** 等，查看每个字段的类型和说明，并**在线试发请求**（注意：大图片 Base64 会很长，浏览器可能卡顿）。

## 四、各交付物「是什么、在哪、怎么看」

### 4.1 可运行服务代码

- **路径**：**`local-service/app.py`**（以及同目录下 `model_wrapper.py`、`rule_based_model.py` 等被引用的模块）。  
- **怎么看**：用 VS Code / Cursor 打开 `local-service` 文件夹；从 `app = FastAPI(...)` 往下看即可看到所有路由与数据模型。

### 4.2 API 文档（含请求/响应示例）

- **中文**：**`help/docs/api-reference.zh.md`**  
- **俄文主文档**：**`help/docs/api-reference.md`**  
- **怎么看**：在编辑器中打开上述 Markdown；文内有 curl 示例、字段表、响应结构说明。  
- **与代码不一致时**：以 **`local-service/app.py`** 中的 Pydantic 模型为准，文档会尽量同步。

### 4.3 演示请求与响应（JSON）

- **路径**：**`reports/deliverables_week4/week-4-demo-request-response.json`**  
- **内容**：包含对 `/generate`、`/refine`、`/edit`、`/render` 的**示例请求体**和**示例响应**；其中 Base64 可能为**截断占位**，便于阅读。  
- **怎么看**：用任意文本编辑器或 JSON 查看器打开即可。  
- **若您需要「真实跑出来的完整 JSON」**：可在服务启动后，用 curl 或 Swagger 调用接口，把返回保存为 `.json`；仓库中 **`reports/week003_*_response.json`** 等文件是历史运行保存的示例，可作格式参考。

- **怎么看**：打开后阅读表格即可：左侧是任务书条款，右侧是仓库路径与简要说明。

## 五、用命令行快速试一次（可选）

在服务已启动的前提下，**最小**的生成请求示例（请把 `image_base64` 换成您自己的短测图或真实 Base64）：

```bash
curl -s -X POST http://localhost:8000/generate \
  -H "Content-Type: application/json" \
  -d '{"image_base64": "iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mP8z8BQDwAEhQGAhKmMIQAAAABJRU5ErkJggg==", "frame_name": "Week4 Demo"}'
```

若成功，终端会打印一段 JSON，其中应包含生成的 **`code`** 等字段。  
更完整的字段（`task_description`、`variables`、`css_hints`）见 **`help/docs/api-reference.zh.md`**。

## 六、和 Figma 插件怎么连（概念说明）

插件侧需要把「当前选中画幅导出为 PNG → Base64」，再 POST 到 **`http://<您的服务地址>:8000/generate`**。  
插件代码与 manifest 中如何配置 `networkAccess`、以及具体消息传递，见仓库 **`figma-plugin/`** 与 **`README.md`**。第四周任务书侧重**服务端**；插件联调可在后续周次继续完善。
