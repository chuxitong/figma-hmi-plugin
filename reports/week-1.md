## Выполненные задачи

### 1. Создание репозитория проекта

Создан репозиторий со следующей структурой:

```
hmi-code-gen/
├── figma-plugin/          # Исходный код плагина Figma
│   ├── manifest.json
│   ├── package.json
│   ├── tsconfig.json
│   └── src/
│       ├── code.ts        # Бэкенд плагина (песочница)
│       └── ui.html        # Интерфейс плагина (iframe)
├── local-service/         # Локальный HTTP-сервис ИИ
├── mockups/               # Промышленные HMI-макеты
├── baseline-tests/        # Результаты тестов базовой модели
├── reports/               # Еженедельные отчёты
└── docs/                  # Техническая документация
```

### 2. Техническая записка по извлечению данных из Figma

Подготовлена документация, описывающая:

**Обязательные данные:**
- Экспорт выбранного фрейма как PNG через `exportAsync()`
- Метаданные фрейма (имя, ширина, высота, фон)
- Обмен сообщениями между логикой плагина и его UI через `postMessage`

**Опциональные данные (улучшение качества):**
- Дизайн-переменные Figma (`figma.variables.*`) — цветовые токены, отступы, типографика
- CSS-подсказки (`getCSSAsync()`) — CSS-свойства для ключевых элементов
- Облегчённая структура дерева узлов — имена, типы, позиции дочерних элементов

### 3. Минимальный прототип (Proof of Concept)

Разработан минимальный работающий плагин Figma:
- **`code.ts`** — обработка выбора пользователя, экспорт фрейма как PNG в base64, извлечение CSS-подсказок и переменных, отправка данных в UI
- **`ui.html`** — интерфейс с кнопками «Generate Code», «Make It Closer to the Mockup», текстовым полем для инструкций редактирования, областью вывода кода и областью предварительного просмотра
- Реализован обмен сообщениями между песочницей и iframe (проверено отправкой ping/pong)

## Текущие проблемы

- Для полного тестирования плагина требуется Figma Desktop с подключённым плагином; пока тестирование проведено через проверку структуры кода.
- API `getCSSAsync()` может быть недоступен в некоторых версиях Figma — добавлена обработка ошибок.
